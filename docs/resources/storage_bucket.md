# mtncloud_storage_bucket (Resource)

Manages an MTN Cloud S3-compatible storage bucket.

## Example Usage

```terraform
resource "mtncloud_storage_bucket" "archive" {
  name        = "my-s3-store"
  bucket_name = "my-archive-bucket"
  access_key  = var.s3_access_key
  secret_key  = var.s3_secret_key
  endpoint    = "https://ps1csp-s3.ict.mtn.com.ng:9021"

  create_bucket = true
}

variable "s3_access_key" {
  type      = string
  sensitive = true
}

variable "s3_secret_key" {
  type      = string
  sensitive = true
}
```

## Schema

### Required

- `access_key` (String, Sensitive) S3 access key.
- `bucket_name` (String) Backing S3 bucket name.
- `endpoint` (String) S3-compatible endpoint URL.
- `name` (String) Unique storage bucket name in MTN Cloud.
- `secret_key` (String, Sensitive) S3 secret key. The API never returns it, so changes here are applied but drift cannot be detected.

### Optional

- `copy_to_store` (Boolean) Copy backups to this store.
- `create_bucket` (Boolean) Create the backing bucket if it does not exist. Defaults to `true`.
- `default_backup_target` (Boolean) Use this bucket as the default backup target.
- `default_deployment_target` (Boolean) Use this bucket as the default deployment target.
- `default_virtual_image_target` (Boolean) Use this bucket as the default virtual image target.
- `retention_policy_days` (Number) Number of days to retain objects before the retention policy applies. Must be >= 1.
- `retention_policy_type` (String) Cleanup mode: `none`, `backup`, or `delete`.
- `retention_provider` (String) Provider used to enforce the retention policy.
- `storage_server` (Number) ID of the storage server backing this bucket.

### Read-Only

- `id` (String) Numeric identifier of the storage bucket.
